SCRIPT: PauseSystem_UMFOSS.cs
AUTHOR: Souvik Kumar (https://github.com/Souvik-Cyclic)
NAMESPACE: GameplayMechanicsUMFOSS.Systems
====================================================

PURPOSE:
A centralised, singleton pause system that freezes gameplay by setting
Time.timeScale to 0, pauses all audio globally, and broadcasts events so
every other system can react without coupling to this one. Supports
configurable pause key, optional auto-pause on application focus loss,
and the store-and-restore timeScale pattern so bullet time and slow motion
survive a pause/resume cycle.

----------------------------------------------------

Code    : using UnityEngine;
Explain : Imports Unity's core namespace. Required for MonoBehaviour, Time,
          Input, AudioListener, KeyCode, SerializeField, and Debug.

Code    : using GameplayMechanicsUMFOSS.Core;
Explain : Imports the framework's Core namespace to access MonoSingletongeneric<T>
          (enforces single instance) and EventBus (decoupled event broadcasting).

Code    : namespace GameplayMechanicsUMFOSS.Systems
Explain : Places this script in the Systems feature group. Keeps pause logic
          separate from Core, Movement, and other feature groups, and prevents
          naming conflicts with other assemblies.

Code    : public class PauseSystem_UMFOSS : MonoSingletongeneric<PauseSystem_UMFOSS>
Explain : Inherits MonoSingletongeneric<T> so only one PauseSystem can ever
          exist at runtime. If a second instance is created (e.g. on scene
          load), the base class calls Destroy(this) on it automatically.
          Pause state is therefore consistent across all scenes without any
          extra DontDestroyOnLoad boilerplate.

Code    : [SerializeField] private KeyCode pauseKey = KeyCode.Escape;
Explain : Exposes the pause key in the Inspector while keeping it private in
          code. Any project can remap it without editing the script — this
          follows the framework principle of putting configuration in data,
          not hardcoded in logic. Defaults to Escape, the most common pause
          key in desktop games.

Code    : [SerializeField] private bool toggleOnFocusLoss = true;
Explain : Controls whether the game auto-pauses when the application window
          loses OS focus (alt-tab, Cmd+Tab, mobile app switch). Desktop games
          should enable this to prevent the player dying off-screen. Mobile
          games should disable it because the OS already suspends the app and
          a double-pause would cause unexpected state.

Code    : [SerializeField] private bool pauseAudio = true;
Explain : When true, sets AudioListener.pause — Unity's single global audio
          switch — which freezes every AudioSource simultaneously and preserves
          exact playback positions. Set false when background music should keep
          playing during pause (e.g. menu music).

Code    : [SerializeField] private bool logStateChanges = false;
Explain : Development-only flag. When true, Debug.Log fires on every Pause()
          and Resume() call showing the stored timeScale. Useful when testing
          bullet-time and pause interaction. Has zero effect on behaviour and
          should be left false in production.

Code    : private float storedTimeScale = 1f;
Explain : Captures Time.timeScale at the moment Pause() is called. On Resume()
          this value is restored exactly — NOT hardcoded to 1.0. This is the
          store-and-restore pattern: if a slow-motion system had set timeScale
          to 0.2 before pause was called, that 0.2 is what comes back on resume.
          Without this, pausing would permanently cancel any active bullet time.

Code    : private bool isPaused = false;
Explain : Tracks current pause state. Used by guard clauses in Pause() and
          Resume() to make both methods idempotent — calling Pause() when
          already paused is a safe no-op and fires no duplicate events.

Code    : public bool IsPaused => isPaused;
Explain : Read-only public property so any system can query pause state without
          subscribing to events. UI elements, AI controllers, and input handlers
          can all poll this directly when they need a one-time check.

Code    : public void Pause()
Explain : Public method to freeze the game. Designed to be called from any
          system — UI buttons, trigger zones, analytics hooks. All callers get
          the same result regardless of call order.

Code    : if (isPaused) return;
Explain : Guard clause. If already paused, exit immediately. Without this, two
          rapid Escape presses would fire two GamePausedEvents and overwrite
          storedTimeScale with 0, making Resume() restore timeScale to 0 and
          permanently freeze the game.

Code    : storedTimeScale = Time.timeScale;
Explain : Captures whatever timeScale is active right now before changing it.
          This is the critical line of the store-and-restore pattern. The value
          could be 0.2 for bullet time, 2.0 for fast-forward — this system
          does not care. It stores and restores whatever was active.

Code    : Time.timeScale = 0f;
Explain : Freezes the simulation. Physics, Animator state machines, particle
          systems, and any Update() or FixedUpdate() code using Time.deltaTime
          all stop. UI using Time.unscaledDeltaTime is unaffected and can still
          animate (e.g. a pause menu sliding in).

Code    : isPaused = true;
Explain : Updates internal state BEFORE publishing the event. If this were set
          after the Publish call, any subscriber that reads IsPaused inside its
          handler would see the wrong value (false instead of true).

Code    : if (pauseAudio) AudioListener.pause = true;
Explain : Pauses all audio via Unity's global audio engine switch. Better than
          AudioListener.volume = 0 (which keeps playback advancing silently)
          and better than calling audioSource.Pause() per-source (which requires
          tracking every dynamically spawned AudioSource). One line, all sources,
          exact playback positions preserved.

Code    : EventBus.Publish(new GamePausedEvent { previousTimeScale = storedTimeScale });
Explain : Broadcasts to all subscribers with the stored timeScale as payload.
          The UI can show the pause menu, AI can disable input, timers can
          freeze — all independently, with zero direct references to this class.
          previousTimeScale lets subscribers (e.g. slow-motion systems) know
          what state to restore when the game resumes.

Code    : public void Resume()
Explain : Public method to unfreeze the game. Prefer calling this directly from
          UI buttons rather than TogglePause(), because TogglePause() from a
          pause-menu button could accidentally re-pause the game if the button
          fires during a transition animation.

Code    : if (!isPaused) return;
Explain : Guard clause. If not paused, exit immediately. Prevents a duplicate
          GameResumedEvent from firing and prevents storedTimeScale from being
          restored to whatever timeScale is currently set.

Code    : Time.timeScale = storedTimeScale;
Explain : Restores exactly the timeScale captured at Pause() time — not 1.0.
          This is where the store-and-restore pattern pays off: slow motion,
          bullet time, and fast-forward all survive a pause/resume cycle without
          any communication between systems.

Code    : isPaused = false;
Explain : Updates internal state BEFORE publishing the event, same reason as
          in Pause() — subscribers reading IsPaused in their handler need to
          see the updated value immediately.

Code    : if (pauseAudio) AudioListener.pause = false;
Explain : Resumes all audio from their exact playback positions. Because
          AudioListener.pause preserves positions (unlike muting), there is no
          audio desync or gap on resume.

Code    : EventBus.Publish(new GameResumedEvent { restoredTimeScale = storedTimeScale });
Explain : Broadcasts to all subscribers. UI hides the pause menu, AI re-enables
          input, timers restart. restoredTimeScale payload lets slow-motion
          systems confirm their timeScale was correctly brought back.

Code    : public void TogglePause()
Explain : Flips between paused and resumed. The only method that should be
          called from keyboard or gamepad input. Centralising the toggle here
          means changing pause-key behaviour requires editing one method only.

Code    : if (isPaused) Resume(); else Pause();
Explain : Routes to the correct method based on current state. Both Pause()
          and Resume() have their own guards so calling them via TogglePause()
          is always safe regardless of current state.

Code    : public float GetPausedTimeScale() => storedTimeScale;
Explain : Returns the timeScale that was stored at the last Pause() call.
          Useful for replay systems, debug overlays, or any system that needs
          to know whether bullet time was active when the game was paused.

Code    : if (Input.GetKeyDown(pauseKey)) TogglePause();
Explain : GetKeyDown fires exactly once per physical key press, not every frame
          the key is held — prevents the toggle from flickering. Routes through
          TogglePause() so all input-driven pause goes through one code path.

Code    : private void OnApplicationFocus(bool hasFocus)
Explain : Unity calls this automatically when the application window gains or
          loses OS focus. This is the correct hook for desktop alt-tab and
          mobile app-switch detection — it is more reliable than checking
          focus state manually in Update().

Code    : if (!toggleOnFocusLoss) return;
Explain : Early exit when auto-pause on focus loss is disabled. Keeps the
          mobile path clean — no state is changed, no events are fired.

Code    : EventBus.Publish(new ApplicationFocusLostEvent());
Explain : Fires the focus-lost event BEFORE calling Pause(). This lets
          analytics or logging systems distinguish "player pressed Escape"
          from "player alt-tabbed" because the event precedes the pause event
          in the same frame.

Code    : if (!isPaused) Pause();
Explain : Auto-pauses only when not already paused. The guard inside Pause()
          would catch a double-call anyway, but checking here avoids emitting
          a redundant ApplicationFocusLostEvent + GamePausedEvent pair.

Code    : EventBus.Publish(new ApplicationFocusGainedEvent());
Explain : Notifies subscribers that the window has regained focus. Systems
          that suspended non-critical work on focus loss can resume it here.

Code    : // Deliberately NOT calling Resume() here.
Explain : Auto-resuming when the window regains focus would start gameplay
          the moment the player clicks back into the window — before they are
          ready. The player must press the pause key deliberately to resume.
          This is the safer and more player-friendly design choice.
